# v13：Router 行为分析（Qwen3-30B-A3B，agent 输入）—— 量化"批内 expert 聚集" idea 的空间

**日期**：2026-07-15
**执行**：GPU 4，HF transformers `output_router_logits`
**目的**：实测 router 在 agent 输入上的行为，量化用户 idea（批内把边缘 token 劝导到已激活专家，减搬运）的可行空间。
**脚本**：`scripts/run_v13_router_analysis.py`

---

## 设置
- 模型：Qwen3-30B-A3B（E=128, topk=8, 48 层）。
- 输入：3 段真实 agent 风格 prompt（工具调用 / 多步规划 / JSON 解析），共 **250 token**，抓全部 48 层的 router logits。
- 测三件事：专家选择分布、router 置信度、topk 权重分布。

---

## 结果一：专家选择分布——高度不均，大量冷门专家
| 指标 | 值 |
|---|---|
| 平均每层"从没被选中"的专家 | **25.5 / 128**（20% 闲置） |
| 热门 25% 专家吃掉的选择占比 | **69.6%** |

- 70% 的 token 挤在 25% 热门专家；20% 专家零流量。
- 中后层更极端（layer 30：45 个专家没被选，热门 25% 占 79%）。
- **→ 流量本就集中，把边缘 token 往热门专家聚集天然阻力小。**

## 结果二：router 置信度——非常低，选择"不坚定"
| 指标 | 值 |
|---|---|
| top1 专家的 softmax 概率 | **0.089**（8.9%） |

- router 对最看好的专家平均只给 8.9% 概率（均匀分布是 1/128=0.78%，只高一个量级）→ 第 1 名与后面分数差距小，router "没把握"。
- **→ 把 token 从第 8 选择换到分数相近的已激活专家，router "意见损失"很小。** 呼应 MS《When Are Experts Misrouted》"很多替代路线不差"。

## 结果三：topk 权重分布——边缘专家贡献很小
一个 token 的 8 个专家归一化权重（跨层平均）：
| rank | 1 | 2 | 3 | 4 | 5 | 6 | 7 | **8** |
|---|---|---|---|---|---|---|---|---|
| 权重(layer20) | 0.233 | 0.167 | 0.138 | 0.117 | 0.102 | 0.090 | 0.080 | **0.072** |

- 平均 rank1 权重 **0.236**，rank8（边缘）**0.071** —— 边缘专家贡献不到 rank1 的 1/3。
- **→ rank7-8 是最安全的"劝导"对象：只占 7% 权重，换掉对输出影响小，但能减少 batch 激活的不同专家数。**

---

## 综合结论：idea 有明确量化空间
| 观察 | 支持点 |
|---|---|
| 20% 专家闲置、70% 流量在 25% 热门专家 | 流量本就集中，聚集阻力小 |
| top1 置信度仅 8.9%（router 不坚定） | 偏离 router 的意见损失小 |
| 边缘(rank8)专家只贡献 7% 权重 | rank7-8 是安全劝导对象 |

**Qwen3-30B-A3B 的 router 在 agent 输入上：既不均衡（可聚集）、又不坚定（可偏离）、边缘专家又不重要（可换）** → 用户的"批内 expert 聚集"idea 有真实、可量化的操作空间。

---

## Caveat
- 本次抓的是 **prefill 前向**的 router 分布；decode 分布可能略有不同，但同模型 router 行为一致，趋势可信。
- 仅 250 token（够看趋势）；严谨结论可扩到数千 token + 多样 agent 数据。

## 下一步（真正验证 idea）
1. **量化聚集收益**：模拟"把每个 token 的 rank7-8 劝导到已激活热门专家"，测 batch 激活专家数 ↓多少 → 搬运 ↓多少。
2. **量化精度代价**：用 counterfactual 方式（借 MS 框架）测这种劝导对 next-token loss / perplexity 的影响，画"搬运节省 vs 精度损失"曲线。
3. 若曲线好（省 30-40% 搬运、精度掉 <1%），就是一个可发表的推理优化点。
